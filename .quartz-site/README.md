# Публикация книги правил

Этот каталог хранит техническую обвязку для публикации книги правил через Quartz.

Публичный читательский исходник для публикации лежит в `../Книга правил v0.4/`.

Перед сборкой Quartz автоматически синхронизирует главы из этой папки в `content/rulebook/` и собирает сайт уже из набора отдельных страниц.

Не редактируйте вручную:

- `content/index.md`
- `content/rulebook/*.md`

Эти файлы пересобираются скриптом `scripts/sync-book.mjs`.

## Команды

- `npm ci` - установить зависимости.
- `npm run sync-book` - пересобрать публикационные страницы из `../Книга правил v0.4/`.
- `npm run build` - синхронизировать главы и собрать сайт.
- `npm run dev` - локальный предпросмотр с автосинхронизацией, если меняются файлы книги.

## UI

- Короткая Quartz-памятка по visual language лежит в `../docs/quartz-ui-style.md`.

## GitHub Pages

Workflow лежит в `../.github/workflows/deploy-rulebook.yml`.

После первого пуша нужно один раз открыть репозиторий на GitHub:

1. `Settings`
2. `Pages`
3. В `Source` выбрать `GitHub Actions`

После этого сайт будет обновляться автоматически при каждом пуше в `master` или `main`, если меняются Quartz-конфиг, workflow или исходники из `../Книга правил v0.4/`.
